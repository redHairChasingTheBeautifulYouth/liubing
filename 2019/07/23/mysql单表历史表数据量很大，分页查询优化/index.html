大数据量分页查询优化

发表于 2019-07-23 | 分类于 mysql | 阅读次数:
字数统计: | 阅读时长 ≈

前言

在公司的历史数据表中，数据量很大，上了百万级别。使用常规的分页查询，不能达到理想的效果，今天在公司测试时，测试结果如下

  • 数据量：3000000以上
  • 花费时间：6845ms

常规分页查询测试

默认按主键id排序，且id为自增

记录行的偏移量为50000
  • 花费时间2786ms
记录行的偏移量为500000
  • 花费时间5980ms
记录行的偏移量为2000000
  • 花费时间15560ms
结论：常规分页查询默认从第一行开始扫描，偏移量越大，查询数据越慢

限定id的起始值优化
